# GitHub Delivery Checks

Before publishing:

1. Run the validator.
2. Confirm exact `SKILL.md` casing.
3. Confirm all referenced templates exist.
4. Confirm README matches the current workflow.
5. Confirm no secrets or machine-specific paths.
6. Confirm a clean Git working tree.
7. Push the intended default branch.
8. Verify the remote repository contains the Skill.
9. Return repository URL, branch, commit, and validation result.
